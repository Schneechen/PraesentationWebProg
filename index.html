Funktionsweise

  • Wörter besteht aus Zeichen, Bsp.: a, á, 啊
  • zusammengefasst in einem Zeichensatz
  • Zeichencode
  • Byte
  • Zeichenkodierung: Schlüssel für die Rückwandlung von Bytes in Zeichencodes

Arten der Zeichenkodierung

  • ASCII
  • ISO 8859-1
  • UTF-16/UTF-8
  • usw...

ASCII

  • American Standard Code for Information Interchange
  • nur elementare englische Zeichen
  • 7-Bit-Zeichenkodierung
  • Grundlage für spätere Kodierungen

UTF-16

  • Kodierung mit variabler Länge für Unicode-Zeichen
  • optimiert für häufig gebrauchten Zeichen
  • Kodierung aller Zeichen in 2 Byte => doppelten Platzbedarf

UTF-8

  • am weitesten verbreitete Kodierung für Unicode-Zeichen
  • MBCS: bis zu 4 Bytes pro Zeichen möglich
  • in den ersten 128 Zeichen deckungsgleich mit ASCII
  • bis zu 3 Folgebytes


Folgebytes UTF-8-Kodierung (binär)
0 0xxxxxxx
1 110xxxxx 10xxxxxx
2 1110xxxx 10xxxxxx 10xxxxxx
3 11110xxx 10xxxxxx 10xxxxxx 10xxxxxx

Möglichkeiten zur Festlegung der Zeichencodierung

BOM: Byte Order Mark

  • Bytereihenfolge-Markierung
  • an den Anfang einer Datei gesetzt
  • gibt an, in welcher Reihenfolge die Bytes ausgewertet werden müssen
  • höchste Priorität

HTTP-Header

  • außerhalb der Datei definiert
  • wird vom Server gesendet
  • Vorteil: gilt für das ganze Dokument
  • Nachteil: bei Änderung - Zugriff auf Server notwendig
  • 2. Priorität

Meta-Header

  • innerhalb der HTML-Datei
  • wirkungslos gegenüber BOM oder HTTP-Header
  • Nachteil: wirkt erst nach einlesen der Meta-Zeile
<!DOCTYPE html><html><head><meta charset="utf-8"/>...
